     合計 11,514 筆。

3. 怎樣才是好的作法？
   - 圖層狀態切換的核心保證來自 `clearLayers()`：每次 `update()` 或
     `startProgressiveLoad()` 切換模式前，先清空所有舊圖層
     （`_heatLayer`/`_markerLayer`/`_fallbackLayer`），再依模式建立新圖層，
     而不是針對個別圖層補丁式地加 `removeLayer`。
   - 驗證統計資料正確性時，優先用「兩個獨立來源算出同一份彙總」互相比對
     （本次：API 預先彙總 vs 前端外部重新聚合全量資料），
     比起單純看數字「感覺合理」更有說服力。

4. 最重要的知識或概念（小學生也能懂）：
   - 「先清乾淨再擺新的」：切換地圖模式時，要先把桌上舊的東西全部收走，
     再擺上新模式要用的東西，不然會兩種東西混在一起。
   - 「卡關超過兩次就換個想法」：同一個問題修了又修還是壞，
     不要再用同一招，要停下來想想是不是一開始的設計就需要改。
   - 「兩種算法對答案」：用兩種不同方式各自算一次同樣的統計數字，
     如果答案一樣，就比較放心資料是對的。

5. 核心的變因是什麼？
   - 行政區分布排名第一名是「萬華區」還是「中山區」，
     取決於查詢時套用的篩選條件（caseType/district/yearFrom/yearTo），
     不同篩選條件下的彙總結果本來就會不同；本次驗證的是「無篩選條件」情境，
     資料與全量聚合一致，代表 `/api/crime/stats` 本身計算正確。

6. 新手可能常犯的誤區？
   - 看到排名跟記憶中不一樣就直覺認為資料錯了，忽略「篩選條件不同」
     也會讓排序改變的可能性。
   - 圖層測試只測「新圖層有沒有建立」，沒測「舊圖層有沒有被移除」，
     導致殘留圖層的 bug 測試覆蓋不到。
